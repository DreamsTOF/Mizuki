# Devices（设备）模块需求文档

## 1. 模块概述

Devices 模块用于管理博客中展示的设备信息，支持按分类（品牌或自定义类别）对设备进行分组展示。每个设备包含名称、图片、规格、描述和外部链接等信息。

## 2. 数据模型

### 2.1 设备（Device）

| 字段名      | 类型   | 说明                     |
| ----------- | ------ | ------------------------ |
| name        | string | 设备名称                 |
| image       | string | 设备图片地址             |
| specs       | string | 设备规格参数             |
| description | string | 设备描述                 |
| link        | string | 设备外部链接（官网等）   |

### 2.2 设备分类（DeviceCategory）

分类以键值对形式组织，键为分类名称（如品牌名或自定义类别名），值为该分类下的设备列表。

## 3. 功能需求

### 3.1 获取设备列表（按分类分组返回）

- 返回所有设备，按分类分组组织。
- 返回数据结构为 `Record<string, Device[]>`。

### 3.2 获取单个设备详情

- 根据设备名称和所属分类，返回单个设备的完整信息。

### 3.3 创建设备

- 支持指定设备所属分类。
- 支持上传设备图片。
- 需收集的数据：名称、图片、规格、描述、外部链接、所属分类。
- 若指定的分类不存在，自动创建该分类。

### 3.4 更新设备

- 支持修改设备的基本信息（名称、图片、规格、描述、外部链接）。
- 支持变更设备所属分类。
- 若变更后的目标分类不存在，自动创建该分类。

### 3.5 删除设备

- 根据设备名称和所属分类删除指定设备。
- 删除后，若该分类下已无任何设备，且该分类不是唯一存在的分类，则自动删除该分类。

### 3.6 获取所有设备分类

- 返回当前所有存在的分类名称列表。
