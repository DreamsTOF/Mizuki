# Diary（日记）模块描述文档

## 1. 模块职责

Diary 模块负责管理用户的生活日记/瞬间（Moment）内容，提供日记内容的展示、筛选、以及后台管理所需的完整数据支持。该模块是 Mizuki 博客的核心内容模块之一，面向访客展示按时间线组织的日记流，同时需要支持后台对日记的增删改查与图片管理。

模块核心职责：
- 提供日记列表的分页查询与多维筛选（标签、日期）。
- 提供单篇日记的完整详情数据。
- 支持日记内容的创建、更新、删除。
- 支持日记图片的独立上传与删除。
- 提供去重后的标签列表，用于前端筛选展示。

## 2. 前端页面与组件结构

### 2.1 页面

| 文件路径 | 说明 |
|---------|------|
| `src/pages/diary.astro` | 日记列表页。展示日记流，支持按标签筛选，显示每篇日记的内容、图片、标签、时间、地点、心情。顶部展示日记总数统计。 |

### 2.2 组件

| 文件路径 | 说明 |
|---------|------|
| `src/components/features/diary/MomentCard.astro` | 单条日记卡片组件。接收单条日记数据，渲染内容文本、图片网格（根据图片数量自适应 1/2/3/4+ 布局）、标签列表、底部元信息（相对时间、地点、心情）。支持图片点击放大（Fancybox）。 |

### 2.3 数据文件

| 文件路径 | 说明 |
|---------|------|
| `src/data/diary.ts` | 前端数据层。定义 `DiaryItem` 接口，提供 `getDiaryList(limit?)` 与 `getAllTags()` 方法。当前为静态数据实现，未来对接后端接口。 |

## 3. 数据模型说明

### 3.1 DiaryItem（日记条目）

```ts
interface DiaryItem {
  id: number;          // 唯一标识
  content: string;     // 日记文本内容
  date: string;        // 创建/发布时间，ISO 8601 格式
  images: string[];    // 图片访问路径数组
  location: string;    // 地点信息
  mood: string;        // 心情描述
  tags: string[];      // 标签数组
}
```

字段说明：

| 字段名 | 类型 | 是否必填 | 说明 |
|-------|------|---------|------|
| id | number | 是 | 日记唯一标识，由后端生成 |
| content | string | 是 | 日记正文内容，纯文本或多行文本 |
| date | string | 是 | ISO 8601 格式日期时间，如 `2025-01-15T10:30:00Z` |
| images | string[] | 否 | 图片访问 URL 数组，空数组表示无图片 |
| location | string | 否 | 地点名称，如 "Shanghai" |
| mood | string | 否 | 心情文本或 Emoji，如 "Happy" 或 "😊" |
| tags | string[] | 否 | 标签字符串数组，用于分类筛选 |

### 3.2 标签（Tag）

标签为日记的附属属性，具有以下特征：
- 每个日记可拥有 0 至多个标签。
- 标签为自由文本字符串，大小写敏感。
- 后端需提供全量去重后的标签列表，供前端筛选器使用。
- 标签与日记为多对多关系（同一标签可属于多篇日记）。

### 3.3 图片（Image）

- 每篇日记可包含 0 至多张图片。
- 前端根据图片数量渲染不同布局：1 张（大图为 400~500px）、2 张（双列）、3 张（首图跨两行 + 右侧两张）、4+ 张（自适应网格，最小 140~160px）。
- 图片通过 `images` 数组中的 URL 访问，支持懒加载。

## 4. 数据流描述

### 4.1 列表页数据流

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   diary.astro   │────▶│  getDiaryList()  │────▶│   后端接口：     │
│   (页面入口)     │     │  (前端数据层)     │     │  获取日记列表    │
└─────────────────┘     └──────────────────┘     └─────────────────┘
         │
         ▼
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  FilterTabs     │◀───▶│  getAllTags()    │◀────│   后端接口：     │
│  (标签筛选器)    │     │  (前端数据层)     │     │  获取标签列表    │
└─────────────────┘     └──────────────────┘     └─────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│                        MomentCard 列表渲染                        │
│  每条日记数据经 MomentCard 组件渲染为卡片，包含：                    │
│  - content（正文）                                                │
│  - images（图片网格，支持 Fancybox 点击放大）                      │
│  - tags（标签列表）                                               │
│  - date → relativeTime（相对时间，如 "3 hours ago"）              │
│  - location（地点）                                               │
│  - mood（心情）                                                   │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 筛选交互数据流

1. 页面加载时，同时请求日记列表与全部标签列表。
2. 标签筛选器（FilterTabs）渲染所有标签及对应日记数量统计。
3. 用户点击标签时，前端通过 `data-tags` 属性对 MomentCard 进行本地显隐过滤（`filtered-out` 类切换）。
4. 无匹配结果时展示空状态提示。

### 4.3 管理后台数据流（未来扩展）

```
┌─────────────────┐     ┌──────────────────────────────────────────┐
│   管理后台页面   │────▶│  创建日记：提交 content/date/location/     │
│                 │     │  mood/tags + 图片上传                      │
└─────────────────┘     └──────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│  更新日记：基于 id 提交完整或部分字段，图片可增删                    │
│  删除日记：基于 id 删除整条记录及关联图片                            │
│  图片管理：独立上传接口返回 URL，独立删除接口清理存储                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 时间展示说明

前端将 `date`（ISO 8601）通过 `formatRelativeTime` 转换为相对时间文本：
- 小于 1 小时：显示 "N minutes ago"
- 小于 24 小时：显示 "N hours ago"
- 大于等于 24 小时：显示 "N days ago"

时间格式化所需的多语言文案由页面传入（`minutesAgo`、`hoursAgo`、`daysAgo`）。
