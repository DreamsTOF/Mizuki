# Diary（日记）模块后端需求文档

## 1. 功能需求列表

| 序号 | 功能点 | 说明 |
|------|--------|------|
| 1 | 获取日记列表 | 支持分页、按标签筛选、按日期范围筛选，按时间倒序返回日记列表 |
| 2 | 获取单篇日记详情 | 根据日记 ID 返回完整的单条日记数据 |
| 3 | 创建日记 | 创建一篇新日记，包含文本、心情、地点、标签、图片关联 |
| 4 | 更新日记 | 基于日记 ID 更新日记内容、心情、地点、标签、图片关联 |
| 5 | 删除日记 | 基于日记 ID 删除整条日记记录及其关联的图片数据 |
| 6 | 上传日记图片 | 支持单张或多张图片上传，返回可访问的图片 URL 列表 |
| 7 | 删除日记图片 | 根据图片 URL 或标识删除已上传的图片文件 |
| 8 | 获取所有日记标签 | 返回所有日记中已使用的去重标签列表，按字母序排列 |

---

## 2. 功能详细需求

### 2.1 获取日记列表

**功能说明**：返回日记列表，默认按时间倒序排列。支持分页及多维筛选。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| page | number | 否 | 页码，从 1 开始，默认 1 | 1 |
| pageSize | number | 否 | 每页条数，默认 10，建议上限 50 | 10 |
| tags | string[] | 否 | 标签筛选，传入多个标签时返回同时包含这些标签的日记 | ["travel", "food"] |
| startDate | string | 否 | 日期范围起始，ISO 8601 格式 | "2025-01-01T00:00:00Z" |
| endDate | string | 否 | 日期范围结束，ISO 8601 格式 | "2025-01-31T23:59:59Z" |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| list | DiaryItem[] | 日记列表，按时间倒序 | 见下方 DiaryItem 结构 |
| total | number | 符合条件的日记总条数 | 128 |
| page | number | 当前页码 | 1 |
| pageSize | number | 每页条数 | 10 |

DiaryItem 结构：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | number | 日记唯一标识 | 1 |
| content | string | 日记正文内容 | "The falling speed of cherry blossoms is five centimeters per second!" |
| date | string | 创建/发布时间，ISO 8601 | "2025-01-15T10:30:00Z" |
| images | string[] | 图片访问 URL 数组 | ["/images/diary/sakura.jpg", "/images/diary/1.webp"] |
| location | string | 地点信息 | "Shanghai" |
| mood | string | 心情描述 | "Happy" |
| tags | string[] | 标签数组 | ["travel", "anime"] |

---

### 2.2 获取单篇日记详情

**功能说明**：根据日记 ID 获取完整的单条日记数据。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| id | number | 是 | 日记唯一标识 | 1 |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | number | 日记唯一标识 | 1 |
| content | string | 日记正文内容 | "The falling speed of cherry blossoms is five centimeters per second!" |
| date | string | 创建/发布时间，ISO 8601 | "2025-01-15T10:30:00Z" |
| images | string[] | 图片访问 URL 数组 | ["/images/diary/sakura.jpg"] |
| location | string | 地点信息 | "Shanghai" |
| mood | string | 心情描述 | "Happy" |
| tags | string[] | 标签数组 | ["travel", "anime"] |

---

### 2.3 创建日记

**功能说明**：创建一篇新日记。图片需先通过上传接口获取 URL，再在创建时传入。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| content | string | 是 | 日记正文内容，允许多行文本 | "Today I visited the park." |
| date | string | 是 | 日记时间，ISO 8601 格式 | "2025-01-15T10:30:00Z" |
| location | string | 否 | 地点信息 | "Shanghai" |
| mood | string | 否 | 心情描述或 Emoji | "😊" |
| tags | string[] | 否 | 标签数组，去重由后端处理 | ["travel", "food"] |
| images | string[] | 否 | 已上传图片的 URL 数组 | ["/images/diary/photo1.jpg"] |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | number | 新创建日记的唯一标识 | 2 |
| content | string | 日记正文内容 | "Today I visited the park." |
| date | string | 日记时间，ISO 8601 | "2025-01-15T10:30:00Z" |
| images | string[] | 图片 URL 数组 | ["/images/diary/photo1.jpg"] |
| location | string | 地点信息 | "Shanghai" |
| mood | string | 心情描述 | "😊" |
| tags | string[] | 标签数组 | ["travel", "food"] |

---

### 2.4 更新日记

**功能说明**：基于日记 ID 更新日记数据。支持全量更新或部分字段更新。图片关联通过传入完整的 images 数组实现替换。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| id | number | 是 | 日记唯一标识 | 1 |
| content | string | 否 | 日记正文内容 | "Updated content here." |
| date | string | 否 | 日记时间，ISO 8601 格式 | "2025-01-15T10:30:00Z" |
| location | string | 否 | 地点信息，传空字符串表示清空 | "Beijing" |
| mood | string | 否 | 心情描述，传空字符串表示清空 | "Excited" |
| tags | string[] | 否 | 完整的标签数组，替换原有标签 | ["life", "daily"] |
| images | string[] | 否 | 完整的图片 URL 数组，替换原有图片 | ["/images/diary/new1.jpg"] |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | number | 日记唯一标识 | 1 |
| content | string | 更新后的正文内容 | "Updated content here." |
| date | string | 更新后的时间 | "2025-01-15T10:30:00Z" |
| images | string[] | 更新后的图片数组 | ["/images/diary/new1.jpg"] |
| location | string | 更新后的地点 | "Beijing" |
| mood | string | 更新后的心情 | "Excited" |
| tags | string[] | 更新后的标签数组 | ["life", "daily"] |

---

### 2.5 删除日记

**功能说明**：根据日记 ID 删除整条日记记录。删除时需同步清理该日记关联的所有图片文件。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| id | number | 是 | 日记唯一标识 | 1 |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| id | number | 已删除日记的标识 | 1 |

---

### 2.6 上传日记图片

**功能说明**：支持单张或多张图片文件上传。上传成功后返回可访问的图片 URL 列表。图片可用于后续创建或更新日记时关联。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| files | File[] | 是 | 图片文件数组，支持单张或多张同时上传 | [sakura.jpg, 1.webp] |

业务约束：
- 支持格式：jpg、jpeg、png、webp、gif
- 单张大小限制：建议不超过 5MB
- 单次上传数量限制：建议不超过 9 张

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| urls | string[] | 上传成功的图片访问 URL 数组，顺序与上传文件一致 | ["/images/diary/sakura.jpg", "/images/diary/1.webp"] |

---

### 2.7 删除日记图片

**功能说明**：根据图片 URL 删除已上传的图片文件。用于在编辑日记时移除不需要的图片。

**输入数据**：

| 字段名 | 类型 | 是否必填 | 说明 | 示例 |
|--------|------|---------|------|------|
| url | string | 是 | 待删除图片的完整访问 URL | "/images/diary/old.jpg" |

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| url | string | 已删除图片的 URL | "/images/diary/old.jpg" |

---

### 2.8 获取所有日记标签

**功能说明**：返回所有日记中已使用的标签去重列表，按字母顺序排列。用于前端筛选器展示。

**输入数据**：无

**输出数据**：

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| tags | string[] | 去重后的标签列表，按字母序 | ["anime", "daily", "food", "life", "travel"] |

## 3. 数据约束与业务规则

### 3.1 ID 生成规则

- 日记 `id` 由后端统一生成，全局唯一。
- 建议使用自增整数或雪花算法等分布式唯一 ID 方案。
- `id` 对外暴露为 number 类型。

### 3.2 日期格式规范

- 所有日期时间字段统一使用 ISO 8601 格式字符串，如 `2025-01-15T10:30:00Z`。
- 后端存储建议统一使用 UTC 时间，前端负责按用户时区展示。
- 日期筛选参数 `startDate` 与 `endDate` 同样使用 ISO 8601 格式。

### 3.3 图片存储路径规范

- 图片上传后应存放于统一的日记图片目录下，建议路径格式：`/images/diary/{filename}`。
- 文件名建议由后端生成，避免原始文件名冲突，可包含时间戳与随机串，如 `20250115_103000_a1b2c3d4.jpg`。
- 返回给前端的 `images` 数组元素为可直接访问的完整 URL 或绝对路径。
- 删除日记时，后端需同步删除该日记 `images` 数组中关联的所有图片物理文件。
- 删除单张图片时，后端需校验该图片是否仍被其他日记引用，避免误删。

### 3.4 标签处理规则

- 标签为自由文本字符串，大小写敏感。
- 创建/更新日记时，后端应对 `tags` 数组进行去重处理（如前端传入 `["a", "a", "b"]`，后端存储为 `["a", "b"]`）。
- 标签字符串应去除首尾空白字符。
- 单条日记的标签数量建议设置上限（如 10 个），单个标签长度建议设置上限（如 30 字符）。
- 空字符串标签应被过滤，不存入数据库。

### 3.5 内容字段约束

- `content` 字段为必填，不允许为空字符串或仅包含空白字符。
- 内容长度建议设置上限（如 5000 字符），防止异常数据。
- 内容中若包含特殊字符，后端应做适当的转义或清洗处理，防止存储层异常。

### 3.6 可选字段默认值

- `images`、`tags`：若未传入或传入空数组，后端应存储为空数组，不返回 null。
- `location`、`mood`：若未传入或传入空字符串，后端可存储为空字符串或 null，返回时统一为空字符串。

### 3.7 列表排序与分页

- 日记列表默认按 `date` 字段降序排列（最新的在前）。
- 分页参数 `page` 从 1 开始计数。
- 若 `pageSize` 超过系统上限，后端应自动截断至上限值。
- 标签筛选传入多个标签时，逻辑为 AND 关系（即日记需同时包含所有指定标签）。

### 3.8 数据一致性要求

- 删除日记时，应保证数据库记录与关联图片文件同时被清理，避免产生孤儿文件。
- 更新日记的 `images` 数组时，若原有图片不在新数组中，后端应评估是否需要物理删除该图片（取决于是否允许多篇日记共用图片）。
- 图片上传与日记创建/更新建议分为独立步骤：先上传图片获取 URL，再创建/更新日记时传入 URL 数组。
