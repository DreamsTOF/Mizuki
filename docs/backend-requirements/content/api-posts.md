# Posts（文章）模块后端 API 需求文档

> **模块名称**: Posts
> **文档版本**: v1.0
> **最后更新**: 2026-05-04
> **关联文档**: [module-posts.md](./module-posts.md)

---

## 1. 概述

本文档定义 Posts 模块后端需提供的 API 功能。每个 API 只描述功能点、输入数据、输出数据，不定义 URL 路径、HTTP 状态码、错误码、认证方式及返回风格。

---

## 2. 数据模型

### 2.1 文章主表（posts）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 主键 | 文章唯一标识 |
| uuid | UUID | 对外暴露的文章标识 |
| title | 字符串 | 文章标题，必填 |
| slug | 字符串 | URL 友好的标识符，唯一 |
| content | 长文本 | Markdown 正文，必填 |
| excerpt | 字符串 | 自动生成的摘要 |
| description | 字符串 | 文章描述/SEO 摘要 |
| author | 字符串 | 作者名 |
| category | 字符串 | 分类 |
| lang | 字符串 | 语言代码 |
| draft | 布尔 | 草稿状态，默认 false |
| pinned | 布尔 | 置顶状态，默认 false |
| priority | 整数 | 置顶优先级，越小越靠前 |
| encrypted | 布尔 | 是否加密，默认 false |
| comment_enabled | 布尔 | 是否启用评论，默认 true |
| password_hash | 字符串 | 加密密码哈希 |
| password_hint | 字符串 | 密码提示 |
| alias | 字符串 | 文章别名，唯一 |
| permalink | 字符串 | 自定义固定链接，唯一 |
| license_name | 字符串 | 许可证名称 |
| license_url | 字符串 | 许可证链接 |
| source_link | 字符串 | 原文链接 |
| cover_image | 字符串 | 封面图片 URL 或路径 |
| published_at | 日期时间 | 发布日期，必填 |
| updated_at | 日期时间 | 更新日期 |
| created_at | 日期时间 | 创建日期，系统自动 |
| view_count | 长整数 | 浏览次数，默认 0 |
| word_count | 整数 | 字数统计 |

### 2.2 标签表（tags）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | 主键 | 标签唯一标识 |
| name | 字符串 | 标签名称，唯一 |
| slug | 字符串 | URL 友好的标签名，唯一 |

### 2.3 文章-标签关联表（post_tags）

| 字段 | 类型 | 说明 |
|------|------|------|
| post_id | 外键 | 关联文章 |
| tag_id | 外键 | 关联标签 |

## 3. API 功能列表

### 3.1 获取文章列表

**功能说明**: 获取文章列表，支持分页、筛选、排序。

**输入数据**:
- 页码（可选，默认 1）
- 每页数量（可选，默认 10，最大 100）
- 是否包含草稿（可选，默认 false）
- 置顶状态筛选（可选）
- 分类筛选（可选）
- 标签筛选（可选）
- 搜索关键词（可选，全文搜索）
- 排序字段（可选，默认发布日期）
- 排序方向（可选，升序/降序）
- 语言筛选（可选）
- 发布日期范围（可选，起始/截止）

**输出数据**:
- 文章列表，每条包含：
  - 文章 ID
  - 标题
  - Slug
  - 描述
  - 作者
  - 分类
  - 标签数组
  - 置顶状态
  - 置顶优先级
  - 草稿状态
  - 加密状态
  - 封面图片路径
  - 发布日期
  - 更新日期
  - 字数统计
  - 文章 URL
- 分页元数据：
  - 当前页码
  - 每页数量
  - 总记录数
  - 总页数
  - 当前排序字段
  - 当前排序方向

**排序规则**:
- 默认排序：置顶文章优先 > 置顶按 priority 升序 > 非置顶按发布日期降序
- 允许通过参数覆盖默认排序

### 3.2 获取单篇文章详情

**功能说明**: 获取单篇文章的完整信息，含 frontmatter 和正文内容。

**输入数据**:
- 文章标识（ID、UUID 或 Slug）

**输出数据**:
- 文章 ID
- 标题
- Slug
- Markdown 正文
- 摘要
- 描述
- 作者
- 分类
- 标签数组
- 语言代码
- 置顶状态
- 置顶优先级
- 草稿状态
- 加密状态
- 是否启用评论
- 封面图片路径
- 许可证名称
- 许可证链接
- 原文链接
- 自定义固定链接
- 别名
- 发布日期
- 更新日期
- 创建日期
- 字数统计
- 浏览次数
- 文章 URL
- 上一篇文章信息（ID、标题、Slug、URL）
- 下一篇文章信息（ID、标题、Slug、URL）

**加密文章特殊处理**:
- 未验证密码时：不返回正文内容，返回加密状态和密码提示
- 已验证密码或管理员：返回完整内容

### 3.3 通过 Slug 获取文章

**功能说明**: 通过 Slug、Alias 或 Permalink 获取单篇文章。

**输入数据**:
- Slug 字符串

**输出数据**:
- 与"获取单篇文章详情"相同

**匹配规则**:
- 优先匹配 slug 字段
- 其次匹配 alias 字段
- 再次匹配 permalink 字段

---

### 3.4 创建文章

**功能说明**: 创建新文章。

**输入数据**:
- 标题（必填）
- Markdown 正文（必填）
- 发布日期（必填）
- 描述
- 标签数组
- 分类
- 作者
- 自定义固定链接
- 置顶状态
- 草稿状态
- 加密状态
- 加密密码（加密文章时必填）
- 密码提示
- 语言代码
- 许可证名称
- 许可证链接
- 原文链接
- 别名
- 是否启用评论
- 封面图片文件（可选）

**输出数据**:
- 创建成功的文章标识
- 标题
- 自动生成的 Slug
- 文章 URL
- 创建时间

**业务规则**:
- 根据标题自动生成 Slug，确保唯一性
- 若指定了 permalink 或 alias，需校验唯一性
- 加密文章必须同时设置密码
- 密码需哈希存储，禁止明文保存
- 封面图片上传后自动转换为 JPG 格式

### 3.5 更新文章

**功能说明**: 更新已有文章的全部或部分字段。

**输入数据**:
- 文章标识（必填）
- 标题（可选）
- Markdown 正文（可选）
- 发布日期（可选）
- 描述（可选）
- 标签数组（可选）
- 分类（可选）
- 作者（可选）
- 自定义固定链接（可选）
- 置顶状态（可选）
- 置顶优先级（可选）
- 草稿状态（可选）
- 加密状态（可选）
- 加密密码（可选）
- 密码提示（可选）
- 语言代码（可选）
- 许可证信息（可选）
- 原文链接（可选）
- 别名（可选）
- 是否启用评论（可选）
- 封面图片文件（可选，传新图则替换，不传则保留原图）

**输出数据**:
- 更新后的完整文章对象

**业务规则**:
- 支持全量更新和部分字段更新
- 修改 permalink 或 alias 时需校验唯一性
- 更新正文时自动更新 updated_at 时间
- 替换封面图片时删除旧封面文件
- 取消加密状态时清除密码哈希

### 3.6 删除文章

**功能说明**: 删除文章及其关联数据。

**输入数据**:
- 文章标识（必填）

**输出数据**:
- 删除成功确认

**业务规则**:
- 删除文章时同步删除关联的标签关联记录
- 删除文章时同步删除封面图片文件
- 可选：清理无关联文章的孤立标签

---

### 3.7 上传文章封面图片

**功能说明**: 为指定文章上传或替换封面图片。

**输入数据**:
- 文章标识（必填）
- 图片文件（必填）

**输出数据**:
- 封面图片访问路径
- 存储位置标识

**处理流程**:
- 验证文件类型（jpg、png、webp、gif）
- 验证文件大小（建议最大 5MB）
- 转换为 JPG 格式（RGBA 转白色背景 RGB）
- 生成唯一文件名
- 存储到对象存储或本地文件系统
- 更新文章的封面图片路径

---

### 3.8 删除文章封面图片

**功能说明**: 删除指定文章的封面图片。

**输入数据**:
- 文章标识（必填）

**输出数据**:
- 删除成功确认

**处理流程**:
- 删除存储系统中的封面图片文件
- 清空文章的封面图片路径字段

**输入数据**: 无

**输出数据**:
- 标签列表，每条包含：
  - 标签 ID
  - 标签名称
  - URL 友好的标签名

### 3.11 获取文章归档

**功能说明**: 按年月分组统计文章，用于归档页面。

**输入数据**:
- 分类筛选（可选）
- 标签筛选（可选）
- 是否包含草稿（可选，默认 false）

**输出数据**:
- 按年份分组的数据，每组包含：
  - 年份
  - 该年文章数量
  - 文章列表，每条包含：
    - 文章 ID
    - 标题
    - 发布日期（完整日期）
    - 文章 URL
    - 标签数组

**排序规则**:
- 年份降序排列
- 同年内文章按发布日期降序排列

---

### 3.12 获取相关文章推荐

**功能说明**: 基于当前文章推荐相关文章。

**输入数据**:
- 当前文章标识（必填）
- 最大返回数量（可选，默认 5）

**输出数据**:
- 相关文章列表，每条包含：
  - 文章 ID
  - 标题
  - Slug
  - 文章 URL
  - 发布日期

**推荐算法**:
- 标签 Jaccard 相似度评分（权重 0-100）
- 标题分词 Jaccard 相似度评分（权重 0-100）
- 时间新鲜度评分（6 个月半衰期指数衰减，权重 0-30）
- 同分类加分（固定 10 分）
- 总分 = 标签相似度 + 标题相似度 + 时间新鲜度 + 分类加分
- 优先返回有标签匹配的文章
- 不足时按时间新鲜度降序补充
- 排除当前文章自身和加密文章

---

### 3.13 获取相邻文章

**功能说明**: 获取当前文章在排序列表中的上一篇和下一篇。

**输入数据**:
- 当前文章标识（必填）

**输出数据**:
- 上一篇文章信息（ID、标题、Slug、URL）
- 下一篇文章信息（ID、标题、Slug、URL）

**计算规则**:
- 基于文章列表排序结果（置顶 > 优先级 > 发布日期）
- 若无上一篇或下一篇，对应字段为空

---

### 3.14 验证加密文章密码

**功能说明**: 验证加密文章的访问密码。

**输入数据**:
- 文章标识（必填）
- 密码（必填）

**输出数据**:
- 验证结果（成功/失败）
- 验证成功时返回：
  - Markdown 正文内容
  - 可选：短期访问令牌，用于后续请求免重复验证

**业务规则**:
- 使用 bcrypt 等安全方式比对密码哈希
- 验证成功后返回正文内容
- 可发放短期令牌避免重复验证
- 验证失败不返回正文

### 3.15 发布草稿 / 设为草稿

- 目标草稿状态（true/false）

**输出数据**:
- 更新后的文章标识和草稿状态

## 4. 业务逻辑规则

### 4.1 Slug 生成规则

1. 移除标题前后空白
2. 转换为小写
3. 将空格、下划线替换为连字符 `-`
4. 移除非字母数字、连字符、Unicode 字符以外的字符
5. 合并连续连字符
6. 截断至 100 个字符
7. 若结果为空，使用 `untitled-{timestamp}`
8. 检查唯一性，若冲突追加 `-{n}`（n 从 2 开始递增）

### 4.2 标签处理规则

1. 标签名去除前后空白
2. 空标签自动过滤
3. 标签不区分大小写存储，但保留原始大小写显示
4. 删除文章时自动清理无关联文章的孤立标签（可选）
5. 单篇文章标签数量建议限制在 10-20 个以内

### 4.3 日期处理规则

| 字段 | 含义 | 可编辑 |
|------|------|--------|
| published_at | 发布日期 | 是 |
| updated_at | 更新日期 | 自动更新（或手动） |
| created_at | 创建日期 | 系统自动 |

- published_at 使用用户指定的日期，默认为创建时的当前日期
- updated_at 在每次内容更新时自动更新为当前时间
- 允许管理员手动设置 updated_at（如迁移旧文章时保留原始更新日期）

### 4.4 草稿与置顶规则

**草稿**:
- 公开列表默认不返回草稿文章
- 单篇获取时未认证用户不可访问草稿
- 草稿文章不参与文章序号计算
- 草稿文章不进入 RSS/Atom 订阅源

**置顶**:
- 置顶文章在列表中优先展示
- 置顶文章按 priority 升序排列（数值越小越靠前）
- priority 为空时排在有数值的后面
- 取消置顶时建议保留 priority 值以便恢复

### 4.5 内容存储方案

- 数据库 TEXT/LONGTEXT 字段存储完整 Markdown 正文
- API 响应时直接返回 Markdown 字符串
- 前端使用 Markdown 渲染器转换为 HTML
- 图片等二进制资源使用独立存储服务

## 5. 数据验证规则

### 5.1 创建/更新文章验证

| 字段 | 规则 |
|------|------|
| 标题 | 必填，1-255 字符 |
| 正文 | 必填，至少 1 字符 |
| Slug | 可选，自动生成；手动指定时需唯一，1-255 字符 |
| 描述 | 可选，最多 500 字符 |
| 发布日期 | 必填，有效日期格式 |
| 标签 | 可选，数组，每项 1-100 字符，最多 20 个 |
| 分类 | 可选，最多 100 字符 |
| 作者 | 可选，最多 100 字符 |
| 优先级 | 可选，非负整数 |
| 自定义固定链接 | 可选，1-255 字符，唯一，格式限制为字母数字下划线连字符 |
| 别名 | 可选，1-255 字符，唯一 |
| 加密密码 | 加密文章必填，6-100 字符 |

### 5.2 封面图片验证

| 规则 | 限制 |
|------|------|
| 文件类型 | jpg、png、webp、gif |
| 文件大小 | 最大 5MB |
| 图片尺寸 | 建议最小 640x360，最大 4096x4096 |
| 文件名 | 自动重命名，禁止特殊字符 |

## 6. 性能优化建议

### 6.1 数据库索引

建议在以下字段或字段组合上建立索引：
- 置顶状态 + 置顶优先级
- 发布日期
- 草稿状态
- 分类
- 语言代码
- Slug（唯一索引）
- Permalink（唯一索引）
- Alias（唯一索引）

### 6.2 缓存策略

| 数据 | 建议缓存时长 |
|------|-------------|
| 文章列表（公开） | 5 分钟 |
| 单篇文章详情 | 10 分钟 |
| 标签列表 | 30 分钟 |
| 分类列表 | 30 分钟 |
| 归档数据 | 10 分钟 |

### 6.3 分页与搜索

- 列表查询最大每页数量限制为 100
- 全文搜索建议使用数据库全文索引或专用搜索引擎
- 支持字段选择，只返回前端需要的字段
