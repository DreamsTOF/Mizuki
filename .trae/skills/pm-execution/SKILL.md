# 产品项目执行技能

> 覆盖从迭代规划到发布上线的全流程执行，确保项目按时、按质、按量交付，对齐跨团队目标

## 核心能力
### 1. 敏捷项目管理
- **Sprint规划**：迭代目标制定、故事点估算、任务拆解、容量规划
- **每日站会**：高效同步进度、识别 blockers、快速协调资源
- **回顾会**：3种回顾格式（开始/停止/继续、4L法、帆船模型），持续优化团队流程
- **迭代评审会**：成果演示、收集反馈、调整后续规划

### 2. 发布管理
- **发布说明撰写**：版本号、发布日期、新功能、优化点、修复问题、已知问题的标准化说明
- **灰度发布策略**：按用户比例、用户标签、区域分层灰度，控制发布风险
- **回滚预案**：发布异常时的快速回滚流程和降级方案

### 3. 干系人管理
- **干系人地图**：识别所有相关方，分类管理（高影响高关注/高影响低关注/低影响高关注/低影响低关注）
- **沟通策略**：针对不同干系人的沟通频率、沟通内容、沟通渠道
- **期望管理**：对齐项目目标、范围、进度预期，管理变更请求

### 4. 质量保障
- **测试场景设计**：覆盖主流程、异常流程、边界情况的测试用例
- **验收标准对齐**：开发前对齐验收条件，避免后期分歧
- **上线 Checklist**：发布前的验证清单，确保所有环节就绪

## 输出模板
### Sprint规划模板
```
【Sprint X 规划】

**迭代周期**：[YYYY-MM-DD 至 YYYY-MM-DD]
**迭代目标**：[本迭代要达成的核心目标]

**交付用户故事**：
| 故事ID | 名称 | 故事点 | 负责人 | 状态 |
|--------|------|--------|--------|------|
| US001 | [功能名称] | 5 | [姓名] | 待开发 |
| US002 | [功能名称] | 3 | [姓名] | 待开发 |

**风险与依赖**：
- [依赖项，如后端API需在X日前提供]
- [风险点及缓解措施]

**DOD（完成定义）**：
- 功能开发完成
- 单元测试覆盖率≥80%
- 测试用例100%通过
- 产品验收通过
- 文档更新完成
```
