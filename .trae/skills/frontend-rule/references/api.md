# API 调用规范

## 核心架构
采用双层函数架构：
- `generated/`：Orval自动生成的代码，禁止手动修改
- `real/`：业务实际使用的函数层，可扩展自定义逻辑

## 核心规则
1. 运行`pnpm api:gen`自动生成API代码
2. 统一从`src/api/real/`导入Controller调用，禁止直接导入generated层
3. 直接使用生成的TypeScript类型，无需二次封装
4. 特殊场景可在real层扩展自定义逻辑

## 拦截器规则
### 请求拦截器自动处理：
- 设备指纹：自动添加`X-Device-Id`请求头
- Token注入：自动添加`satoken`请求头
- 请求加密：非白名单接口自动AES加密请求体

### 响应拦截器自动处理：
- 统一错误提示：业务失败自动弹出ElMessage.error
- 响应解密：自动解密加密响应数据
- 业务码处理：40100登录失效、40101密钥过期自动跳转
- 数据提取：自动返回`response.data.data`，业务代码直接使用

## 错误处理
- 默认由拦截器统一弹窗提示错误
- 特殊场景可通过配置自定义覆盖错误提示

## 其他规则
- 统一使用`useRequest`Hook管理loading状态
- 组件卸载时默认自动取消未完成请求，特殊场景可配置跳过
- 核心模型数据缓存到Pinia Store，业务请求按需使用缓存Hook
