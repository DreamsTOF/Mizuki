# 需求优先级与目标管理技能

> 6大优先级评估框架，数据驱动需求排序，确保资源投入到最高价值的功能上

## 核心能力
### 1. 优先级评估框架
- **KANO模型**：需求分类（基础型/期望型/兴奋型/无差异型），指导资源分配策略
- **RICE评分法**：(触达范围×影响力×置信度)/工作量，规模化需求排序
- **ICE评分法**：影响力×置信度×简易度，快速需求排序
- **MoSCoW法则**：Must/Should/Could/Won't四级分类，敏捷项目范围管理
- **机会评分法**：重要性 × (1 − 满意度)，识别最高价值的用户痛点
- **价值-工作量矩阵**：2×2快速分类，优先选择高价值低工作量的需求

### 2. 目标管理
- **OKR制定**：鼓舞人心的目标 + 可衡量的关键成果，对齐团队方向
- **北极星指标设计**：产品核心价值的量化体现，指导长期发展
- **OMTM（唯一最重要指标）**：每个阶段聚焦一个核心指标，避免分散精力

### 3. 需求池管理
- 需求分类标准
- 版本规划方法
- 需求变更管理流程

## 输出模板
### RICE优先级评估表
```
【需求优先级评估表】

| 需求名称 | 触达范围(R) | 影响力(I) | 置信度(C) | 工作量(E) | RICE得分 | 优先级 |
|---------|------------|----------|----------|----------|---------|--------|
| 需求A | 10000用户 | 3（高影响） | 80% | 2人月 | (10000×3×0.8)/2=12000 | P0 |
| 需求B | 5000用户 | 2（中影响） | 90% | 1人月 | (5000×2×0.9)/1=9000 | P1 |
| 需求C | 2000用户 | 1（低影响） | 70% | 0.5人月 | (2000×1×0.7)/0.5=2800 | P2 |
```

### OKR模板
```
【OKR制定】

**目标（Objective）**：[鼓舞人心的定性目标，比如"让新用户享受轻松愉快的引导体验"]

**关键成果（Key Results）**：
- KR1：[可量化成果，如"引导调研CSAT评分 >= 75%"]
- KR2：[可量化成果，如"66%以上的用户在两天内完成引导"]
- KR3：[可量化成果，如"平均首次价值实现时间(TTV) <= 20分钟"]
```

### MoSCoW范围划分模板
```
【迭代范围定义】

**Must Have（必须有，60%）**：
- [不做就无法交付的核心功能]

**Should Have（应该有，20%）**：
- [重要但不影响核心闭环的功能]

**Could Have（可以有，15%）**：
- [有了更好，没有也可以接受的功能]

**Won't Have（本次不做，5%）**：
- [本次迭代明确不做的功能，留待后续版本]
```
