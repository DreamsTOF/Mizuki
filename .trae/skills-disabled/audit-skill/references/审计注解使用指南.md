# 审计注解使用指南

## 一、注解概览

Dreamtof 审计系统提供三个核心注解：

| 注解 | 目标 | 用途 |
|------|------|------|
| `@AuditLog` | 方法 | 标记需要审计的 Service 方法 |
| `@AuditEntityId` | 字段 | 标记 DTO 中的实体 ID 字段 |
| `@AuditReference` | 字段 | 标记外键字段，用于自动翻译 |

---

## 二、@AuditLog 详解

### 2.1 基本用法

```java
@Service
@RequiredArgsConstructor
public class SongAppService {

    private final SongRepository songRepository;
    private final SmartTransactionHelper tx;

    // 基础用法：单实体
    @AuditLog(entity = SongPO.class)
    public void updateSong(SongUpdateReq req) {
        tx.execute(() -> {
            SongPO song = songRepository.selectOneById(req.getId());
            Assembler.toEntity(req, song);
            songRepository.update(song);
        });
    }
}
```

**生成的审计日志**：
```json
{
  "logType": "AUDIT_LOG",
  "module": "SongAppService",
  "action": "updateSong",
  "entity": "SongPO",
  "entityId": "uuid-xxx",
  "operation": "UPDATE",
  "description": "更新SongPO(ID:uuid-xxx)：【title】从\"晴天\"改为\"晴天(Live)\"；【artist】从\"周杰伦\"改为\"Jay Chou\"",
  "changes": [...]
}
```

### 2.2 属性详解

#### `module` - 业务模块

**作用**：标识业务模块名称，用于日志分类

**取值策略**：
1. 显式指定：`module = "歌曲管理"`
2. 留空继承：自动从 Controller 类的 `@Tag` 注解获取
3. 兜底：使用类名（`SongAppService`）

```java
// 方案1：显式指定
@AuditLog(entity = SongPO.class, module = "歌曲管理")
public void createSong(...) { ... }

// 方案2：继承 Controller Tag（推荐）
@Tag(name = "歌曲管理")  // Controller 类上
@RestController
public class SongController { ... }

// Service 方法无需指定 module，自动继承 "歌曲管理"
@AuditLog(entity = SongPO.class)  // module 自动为 "歌曲管理"
public void createSong(...) { ... }
```

#### `action` - 业务动作

**作用**：描述具体操作，如"创建歌曲"、"更新状态"等

**取值策略**：
1. 显式指定：`action = "批量导入歌曲"`
2. 留空继承：自动从方法的 `@Operation` 注解获取 `summary`
3. 兜底：使用方法名（`batchImportSongs`）

```java
// 方案1：显式指定
@AuditLog(entity = SongPO.class, action = "批量导入歌曲")
public void batchImport(List<SongImportDTO> list) { ... }

// 方案2：继承 Swagger Operation（推荐）
@Operation(summary = "批量导入歌曲")  // Controller 方法上
@PostMapping("/songs/batch-import")
public void batchImport(@RequestBody List<SongImportDTO> list) { ... }

// Service 方法无需指定 action，自动继承 "批量导入歌曲"
@AuditLog(entity = SongPO.class)  // action 自动为 "批量导入歌曲"
public void batchImport(List<SongImportDTO> list) { ... }
```

#### `entity` vs `entities` - 实体配置

**单实体模式**（`entity`）：
```java
// 旧代码兼容
@AuditLog(entity = SongPO.class)
public void updateSong(SongUpdateReq req) { ... }
```

**多实体模式**（`entities`，推荐）：
```java
// 一次操作涉及多个实体
@AuditLog(
    entities = {SongPO.class, SongLabelMapPO.class},
    module = "标签管理",
    action = "设置歌曲标签"
)
public void setSongLabels(UUID songId, List<UUID> labelIds) {
    tx.execute(() -> {
        // 删除旧关联
        mapRepository.deleteBySongId(songId);

        // 插入新关联
        labelIds.forEach(labelId -> {
            SongLabelMapPO map = new SongLabelMapPO();
            map.setSongId(songId);
            map.setLabelId(labelId);
            mapRepository.insert(map);
        });
    });
}
```

**生成的审计日志**（批量）：
```json
{
  "logType": "AUDIT_LOG",
  "module": "标签管理",
  "action": "设置歌曲标签",
  "entity": "批量操作(SongPO,SongLabelMapPO)",
  "entityId": "多个主键",
  "operation": "BATCH",
  "description": "批量操作 (共2项变更)：\n(1) 删除SongLabelMapPO(ID:xxx)\n(2) 创建SongLabelMapPO(ID:yyy)",
  "changes": [...]
}
```

#### `partial` - 简略模式

**作用**：控制是否记录字段级变更详情

**取值**：
- `AuditOption.TRUE`：简略模式（仅记录摘要）
- `AuditOption.FALSE`：完整模式（记录所有字段变更）
- `AuditOption.DEFAULT`：跟随全局配置 `defaultPartial`

```java
// 强制简略模式（大批量操作）
@AuditLog(
    entity = SongPO.class,
    partial = AuditOption.TRUE
)
public void batchUpdateStatus(List<UUID> ids, Integer status) {
    // 仅记录："更新SongPO(ID:xxx) [简略模式]"
    // 不记录具体字段变更（节省内存）
}

// 强制完整模式（重要操作）
@AuditLog(
    entity = SongPO.class,
    partial = AuditOption.FALSE
)
public void updateSongPrice(UUID songId, BigDecimal newPrice) {
    // 记录所有字段变更详情
}
```

**简略模式输出示例**：
```json
{
  "entity": "SongPO",
  "entityId": "uuid-xxx",
  "operation": "UPDATE",
  "description": "更新SongPO(ID:uuid-xxx) [简略模式]",
  "changes": []  // 空列表
}
```

#### `skipNull` - 跳过空字段

**作用**：对比字段时是否跳过新值为 null 的字段

**取值**：
- `AuditOption.TRUE`：跳过新值为 null 的字段（不记录"字段被置空"）
- `AuditOption.FALSE`：记录新值为 null 的字段变更
- `AuditOption.DEFAULT`：跟随全局配置 `defaultSkipNull`（默认 true）

```java
// 记录字段被置空
@AuditLog(
    entity = SongPO.class,
    skipNull = AuditOption.FALSE
)
public void clearSongNotes(UUID songId) {
    SongPO song = songRepository.selectOneById(songId);
    song.setNotes(null);  // 会记录：【备注】被置空(原值"xxx")
    songRepository.update(song);
}

// 跳过空字段（默认）
@AuditLog(entity = SongPO.class)  // skipNull = DEFAULT → true
public void clearSongNotes(UUID songId) {
    // 不会记录 notes 字段的变更（因为新值为 null）
}
```

#### `systemFields` - 系统字段

**作用**：是否记录系统字段（如 `createTime`、`updateTime`、`version`）

**取值**：
- `AuditOption.TRUE`：记录系统字段
- `AuditOption.FALSE`：不记录系统字段
- `AuditOption.DEFAULT`：跟随全局配置 `defaultSystemFields`（默认 false）

```java
// 记录系统字段变更
@AuditLog(
    entity = SongPO.class,
    systemFields = AuditOption.TRUE
)
public void updateSong(SongUpdateReq req) {
    // 会记录 updateTime、version 的变更
}

// 不记录系统字段（默认）
@AuditLog(entity = SongPO.class)  // systemFields = DEFAULT → false
public void updateSong(SongUpdateReq req) {
    // 不会记录 updateTime、version 的变更
}
```

#### `refresh` - 回填快照

**作用**：是否在事务结束前回填数据库最新快照

**取值**：
- `AuditOption.TRUE`：回填（事务提交前执行 SELECT 获取最新状态）
- `AuditOption.FALSE`：不回填（直接使用内存中的对象）
- `AuditOption.DEFAULT`：跟随全局配置 `defaultRefresh`（默认 true）

**回填场景**：
1. **触发器修改**：数据库触发器自动更新字段（如 `update_time`）
2. **幻读防护**：其他事务并发修改同一行数据
3. **默认值填充**：数据库默认值（如 `version` 自增）

```java
// 回填最新快照（推荐）
@AuditLog(
    entity = SongPO.class,
    refresh = AuditOption.TRUE
)
public void updateSong(SongUpdateReq req) {
    tx.execute(() -> {
        SongPO song = songRepository.selectOneById(req.getId());
        Assembler.toEntity(req, song);
        songRepository.update(song);
        // 事务提交前会重新 SELECT 一次，获取数据库最新状态
        // 包括：触发器修改、并发修改、默认值
    });
}

// 不回填（性能优先）
@AuditLog(
    entity = SongPO.class,
    refresh = AuditOption.FALSE
)
public void updateSong(SongUpdateReq req) {
    // 直接使用内存中的对象作为 newState
    // 性能最高，但可能遗漏数据库侧变更
}
```

### 2.3 完整示例

```java
@Service
@RequiredArgsConstructor
public class SongAppService {

    private final SongRepository songRepository;
    private final SmartTransactionHelper tx;

    /**
     * 批量导入歌曲（完整配置示例）
     */
    @AuditLog(
        entities = {SongPO.class, SongLabelMapPO.class},
        module = "歌曲管理",          // 显式指定模块
        action = "批量导入歌曲",      // 显式指定动作
        partial = AuditOption.TRUE,   // 强制简略模式（大批量）
        skipNull = AuditOption.TRUE,  // 跳过空字段
        systemFields = AuditOption.FALSE,  // 不记录系统字段
        refresh = AuditOption.FALSE  // 不回填（性能优先）
    )
    public void batchImport(List<SongImportDTO> importList) {
        tx.execute(() -> {
            importList.forEach(dto -> {
                SongPO song = Assembler.toEntity(dto);
                songRepository.insert(song);

                // 插入标签关联
                if (dto.getLabelIds() != null) {
                    dto.getLabelIds().forEach(labelId -> {
                        SongLabelMapPO map = new SongLabelMapPO();
                        map.setSongId(song.getId());
                        map.setLabelId(labelId);
                        mapRepository.insert(map);
                    });
                }
            });
        });
    }
}
```

## 三、@AuditEntityId 详解

### 3.1 基本用法

**作用**：标记 DTO 中代表实体 ID 的字段，供审计切面精准抓取

```java
// DTO 定义
public class SongUpdateReq {

    @AuditEntityId(target = SongPO.class)  // 标记 ID 字段
    private UUID songId;

    @Check(required = true, msg = "歌曲名称")
    private String title;

    @Check(required = true, msg = "歌手")
    private String artist;

    // 其他字段...
}

// Service 方法
@AuditLog(entity = SongPO.class)
public void updateSong(@Valid SongUpdateReq req) {
    // 切面自动识别 @AuditEntityId 字段
    // 提取 songId，查询 SongPO 旧状态
    SongPO song = songRepository.selectOneById(req.getSongId());
    Assembler.toEntity(req, song);
    songRepository.update(song);
}
```

### 3.2 属性详解

#### `target` - 目标实体类

**作用**：指定该 ID 字段对应的实体类

**取值**：
- 显式指定：`target = SongPO.class`
- 留空（`void.class`）：自动推断（通过 `@AuditLog` 的 `entity` 或 `entities`）

```java
// 方案1：显式指定（推荐）
public class SongUpdateReq {
    @AuditEntityId(target = SongPO.class)
    private UUID songId;
    // ...
}

// 方案2：留空自动推断
@AuditLog(entity = SongPO.class)  // 上下文已知实体
public void updateSong(@Valid SongUpdateReq req) {
    // DTO 中的 ID 字段自动推断为 SongPO 的 ID
}

public class BatchUpdateReq {
    @AuditEntityId  // target = void.class（默认）
    private UUID id;  // 自动匹配 @AuditLog 中的 entities
}
```

### 3.3 高级用法

**集合类型 ID**：
```java
public class BatchDeleteReq {

    @AuditEntityId(target = SongPO.class)
    private List<UUID> songIds;  // 支持集合

    // 审计时会批量查询所有 ID 对应的实体
}
```

**嵌套 DTO**：
```java
public class SongBatchUpdateReq {

    @AuditEntityId(target = SongPO.class)
    private UUID songId;

    private List<LabelUpdateDTO> labels;  // 嵌套对象

    public static class LabelUpdateDTO {
        @AuditEntityId(target = SongLabelMapPO.class)
        private UUID mapId;  // 嵌套 ID 也会被识别

        private String labelName;
    }
}
```

## 四、@AuditReference 详解

### 4.1 基本用法

**作用**：标记外键字段，审计时自动将 ID 翻译为业务名称

```java
@Table("song")
public class SongPO {

    @Id
    private UUID id;

    @Column("user_id")
    private UUID userId;

    @Column("label_id")
    private UUID labelId;

    // 审计时自动翻译 userId 为用户名
    @AuditReference(
        target = UserPO.class,   // 目标实体
        label = "username",      // 展示字段
        idCol = "id"            // 主键列名
    )
    public UUID getUserId() {
        return userId;
    }

    // 审计时自动翻译 labelId 为标签名
    @AuditReference(
        target = SongLabelPO.class,
        label = "name",
        idCol = "id"
    )
    public UUID getLabelId() {
        return labelId;
    }
}
```

**审计日志输出**：
```json
{
  "changes": [
    {
      "field": "userId",
      "name": "用户",
      "old": "123e4567-e89b-12d3-a456-426614174000",
      "new": "张三"  ← 自动翻译！
    },
    {
      "field": "labelId",
      "name": "标签",
      "old": "uuid-label-1",
      "new": "热门"  ← 自动翻译！
    }
  ]
}
```

### 4.2 属性详解

#### `target` - 目标实体类

**作用**：指定外键引用的目标实体

**要求**：
- 目标实体必须在 MyBatis-Flex 中正确映射（`@Table` 注解）
- 必须有主键（`@Id` 注解）

```java
@AuditReference(
    target = UserPO.class,  // ✅ UserPO 必须有 @Table 和 @Id
    label = "username",
    idCol = "id"
)
public UUID getUserId() {
    return userId;
}
```

#### `label` - 展示字段名

**作用**：指定目标实体中用于展示的字段

**常见值**：
- 用户表：`username`、`nickname`、`realName`
- 标签表：`name`
- 分类表：`categoryName`

```java
@AuditReference(
    target = UserPO.class,
    label = "nickname",  // 使用 nickname 字段作为展示名
    idCol = "id"
)
public UUID getUserId() {
    return userId;
}
```

#### `idCol` - 主键列名

**作用**：指定目标表的主键列名（默认为 `"id"`）

**使用场景**：当主键列名不是 `id` 时需显式指定

```java
@Table("user")
public class UserPO {

    @Id
    @Column("user_id")  // 列名为 user_id
    private UUID userId;

    @Column("user_name")
    private String username;
}

// 引用时需指定 idCol
@AuditReference(
    target = UserPO.class,
    label = "username",
    idCol = "user_id"  // ✅ 指定实际列名
)
public UUID getUserId() {
    return userId;
}
```

### 4.3 集合类型外键

```java
@Table("song")
public class SongPO {

    @Column("tag_ids")
    private List<UUID> tagIds;  // 多个标签 ID

    @AuditReference(
        target = SongLabelPO.class,
        label = "name",
        idCol = "id"
    )
    public List<UUID> getTagIds() {
        return tagIds;
    }
}
```

**审计日志输出**：
```json
{
  "changes": [
    {
      "field": "tagIds",
      "name": "标签",
      "old": ["uuid-1", "uuid-2"],
      "new": ["中文", "热门"]  ← 自动翻译为列表
    }
  ]
}
```

### 4.4 容错降级

**设计**：如果目标表不存在或查询失败，自动回退显示原始 ID

```java
@AuditReference(
    target = NonExistentPO.class,  // 不存在的实体
    label = "name",
    idCol = "id"
)
public UUID getSomeId() {
    return someId;
}

// 输出：{"old": "uuid-xxx", "new": "uuid-xxx(未知)"}
```

## 五、组合使用示例

### 5.1 完整 Service 方法

```java
@Service
@RequiredArgsConstructor
public class SongAppService {

    private final SongRepository songRepository;
    private final SmartTransactionHelper tx;

    /**
     * 更新歌曲信息（完整注解配置）
     */
    @AuditLog(
        entity = SongPO.class,
        module = "歌曲管理",          // 模块名
        action = "更新歌曲信息",      // 动作描述
        partial = AuditOption.FALSE,  // 完整模式
        skipNull = AuditOption.TRUE,  // 跳过空字段
        systemFields = AuditOption.FALSE,  // 不记录系统字段
        refresh = AuditOption.TRUE    // 回填最新快照
    )
    public void updateSong(@Valid SongUpdateReq req) {
        tx.execute(() -> {
            SongPO song = songRepository.selectOneById(req.getSongId());
            Asserts.isNotNull(song, "歌曲不存在");

            // 字段赋值
            song.setTitle(req.getTitle());
            song.setArtist(req.getArtist());
            song.setCoverUrl(req.getCoverUrl());
            song.setAlbum(req.getAlbum());
            song.setDuration(req.getDuration());
            song.setLanguage(req.getLanguage());
            song.setNotes(req.getNotes());
            song.setStatus(req.getStatus());

            // 更新
            songRepository.update(song);
        });
    }
}
```

### 5.2 DTO 完整配置

```java
public class SongUpdateReq {

    @AuditEntityId(target = SongPO.class)
    @Check(required = true, msg = "歌曲 ID")
    private UUID songId;

    @Check(required = true, msg = "歌曲名称")
    private String title;

    @Check(required = true, msg = "歌手")
    private String artist;

    @Check(url = true, msg = "封面 URL")
    private String coverUrl;

    private String album;

    @Check(min = 0, msg = "时长")
    private Integer duration;

    @Check(required = true, msg = "语种")
    private String language;

    @AuditReference(
        target = UserPO.class,
        label = "username",
        idCol = "id"
    )
    @Check(required = true, msg = "创建人")
    private UUID createdBy;

    @AuditReference(
        target = SongLabelPO.class,
        label = "name",
        idCol = "id"
    )
    private List<UUID> labelIds;  // 多对多标签

    @Check(enumClass = SongStatus.class, msg = "状态")
    private String status;

    private String notes;
}
```

### 5.3 批量操作配置

```java
@Service
@RequiredArgsConstructor
public class BatchOperationService {

    private final SmartTransactionHelper tx;

    /**
     * 批量更新歌曲状态（1000 条）
     * 使用简略模式避免内存溢出
     */
    @AuditLog(
        entity = SongPO.class,
        module = "歌曲管理",
        action = "批量更新状态",
        partial = AuditOption.TRUE,   // ✅ 简略模式
        refresh = AuditOption.FALSE   // ✅ 不回填（性能）
    )
    public void batchUpdateStatus(List<UUID> songIds, SongStatus newStatus) {
        // 分批处理（每批 200 条）
        Lists.partition(songIds, 200).forEach(batch -> {
            tx.execute(() -> {
                batch.forEach(id -> {
                    SongPO song = new SongPO();
                    song.setId(id);
                    song.setStatus(newStatus);
                    songRepository.update(song);
                });
            });
        });
    }

    /**
     * 批量删除（软删除）
     */
    @AuditLog(
        entity = SongPO.class,
        module = "歌曲管理",
        action = "批量删除歌曲",
        partial = AuditOption.TRUE
    )
    public void batchDelete(List<UUID> songIds) {
        tx.execute(() -> {
            songIds.forEach(id -> {
                SongPO song = new SongPO();
                song.setId(id);
                song.setDeleted(true);  // 逻辑删除
                songRepository.update(song);
            });
        });
    }
}
```

## 六、配置继承与覆盖

### 6.1 配置优先级

```
方法注解 (@AuditLog) > 全局配置 (application.yml)
```

**优先级规则**：
1. 如果 `@AuditLog` 中属性显式指定（非 `DEFAULT`），使用方法值
2. 如果 `@AuditLog` 中为 `AuditOption.DEFAULT`，继承全局配置
3. 如果全局配置未设置，使用硬编码默认值

```yaml
# application.yml
dreamtof:
  auto-audit:
    defaultPartial: false        # 默认完整模式
    defaultSkipNull: true        # 默认跳过空字段
    defaultSystemFields: false   # 默认不记录系统字段
    defaultRefresh: true         # 默认回填快照
```

```java
// 场景1：完全继承全局配置
@AuditLog(entity = SongPO.class)  // 所有属性均为 DEFAULT
public void updateSong(...) { ... }
// 实际效果：partial=false, skipNull=true, systemFields=false, refresh=true

// 场景2：部分覆盖
@AuditLog(
    entity = SongPO.class,
    partial = AuditOption.TRUE  // 覆盖全局，强制简略模式
    // 其他属性仍继承全局
)
public void batchUpdate(...) { ... }
// 实际效果：partial=true, skipNull=true, systemFields=false, refresh=true

// 场景3：全部显式指定
@AuditLog(
    entity = SongPO.class,
    partial = AuditOption.FALSE,
    skipNull = AuditOption.FALSE,
    systemFields = AuditOption.TRUE,
    refresh = AuditOption.FALSE
)
public void criticalUpdate(...) { ... }
// 实际效果：全部使用指定值，不继承全局
```

### 6.2 推荐配置策略

**全局配置（application.yml）**：
```yaml
dreamtof:
  auto-audit:
    enabled: true
    defaultPartial: false        # 默认完整模式
    defaultSkipNull: true        # 默认跳过空字段（减少噪音）
    defaultSystemFields: false   # 默认不记录系统字段
    defaultRefresh: true         # 默认回填（确保数据准确）
```

**方法注解策略**：
- **普通 CRUD**：不指定任何属性（继承全局）
- **大批量操作**：`partial = AuditOption.TRUE` + `refresh = AuditOption.FALSE`
- **关键操作**：`partial = AuditOption.FALSE`（强制完整模式）
- **特殊场景**：按需覆盖 `skipNull`、`systemFields`

## 七、最佳实践清单

### ✅ 必须遵守

1. **所有写操作必须添加 `@AuditLog`**
   ```java
   @Service
   public class SongAppService {
       @AuditLog(entity = SongPO.class)  // ✅
       public void createSong(...) { ... }

       @AuditLog(entity = SongPO.class)  // ✅
       public void updateSong(...) { ... }

       @AuditLog(entity = SongPO.class)  // ✅
       public void deleteSong(...) { ... }
   }
   ```

2. **实体类必须使用 MyBatis-Flex 注解**
   ```java
   @Table("song")  // ✅
   public class SongPO {
       @Id  // ✅
       private UUID id;

       @Column("title")  // ✅
       private String title;
   }
   ```

3. **DTO 使用 `@AuditEntityId` 标记 ID**
   ```java
   public class SongUpdateReq {
       @AuditEntityId(target = SongPO.class)  // ✅
       private UUID songId;
   }
   ```

4. **外键字段使用 `@AuditReference`**
   ```java
   @AuditReference(
       target = UserPO.class,
       label = "username",
       idCol = "id"
   )
   public UUID getUserId() { ... }  // ✅
   ```

5. **事务使用 `SmartTransactionHelper`**
   ```java
   @Service
   public class SongAppService {
       private final SmartTransactionHelper tx;

       public void updateSong(...) {
           tx.execute(() -> {  // ✅
               // 业务逻辑
           });
       }
   }
   ```

### ❌ 禁止行为

1. **禁止手动编写审计日志**
   ```java
   // ❌ 错误：手动记录审计
   public void updateSong(...) {
       auditLogService.create(...);
       // 业务逻辑
   }

   // ✅ 正确：使用注解自动审计
   @AuditLog(entity = SongPO.class)
   public void updateSong(...) { ... }
   ```

2. **禁止忽略事务异常**
   ```java
   // ❌ 错误：吞没异常导致事务不提交
   @AuditLog(entity = SongPO.class)
   public void updateSong(...) {
       try {
           // 业务逻辑
       } catch (Exception e) {
           log.error("失败", e);
           // 异常被吞没，事务回滚，审计日志不生成
       }
   }

   // ✅ 正确：异常向上抛出
   @AuditLog(entity = SongPO.class)
   public void updateSong(...) {
       // 业务逻辑（异常自动传播）
   }
   ```

3. **禁止大批量操作不优化**
   ```java
   // ❌ 错误：10000 条数据单次事务，必 OOM
   @AuditLog(entity = SongPO.class)
   public void batchImport10000(List<SongPO> list) {
       songRepository.insert(list);  // 10000 条单次事务
   }

   // ✅ 正确：分批 + 简略模式
   @AuditLog(entity = SongPO.class, partial = AuditOption.TRUE)
   public void batchImport10000(List<SongPO> list) {
       Lists.partition(list, 200).forEach(batch -> {
           tx.execute(() -> songRepository.insert(batch));
       });
   }
   ```

4. **禁止使用 `@Transactional`**
   ```java
   // ❌ 错误：@Transactional 与审计上下文不同步
   @Service
   public class SongAppService {
       @Transactional  // ❌
       @AuditLog(entity = SongPO.class)
       public void updateSong(...) { ... }
   }

   // ✅ 正确：使用 SmartTransactionHelper
   @Service
   public class SongAppService {
       private final SmartTransactionHelper tx;

       @AuditLog(entity = SongPO.class)
       public void updateSong(...) {
           tx.execute(() -> {  // ✅
               // 业务逻辑
           });
       }
   }
   ```
