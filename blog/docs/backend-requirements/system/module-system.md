# 系统通用模块 — 系统架构与模块划分

> 本文档描述 Mizuki 博客项目中与具体业务模块无关的通用系统能力，包括站点配置、About 页面、文件上传、OG 图片生成、RSS/Atom 订阅、站点统计等。为前后端分离改造提供模块边界与数据需求说明。

---

## 1. 模块定位

系统通用模块承载所有业务模块共享的基础设施能力，不依赖特定业务数据，但被所有业务模块依赖。当前前端通过构建时读取本地文件获取这些数据，后端化后需通过统一接口提供。

---

## 2. 功能模块清单

### 2.1 站点配置管理

**功能说明**

站点全局配置的集中存储与分发。当前所有配置硬编码在 `src/config.ts` 中，前端组件直接 import 使用。后端化后，配置需持久化存储，并在构建时通过接口获取。

**配置分组**

| 配置组 | 说明 | 典型配置项 |
|--------|------|-----------|
| 站点元信息 | 站点基础标识 | 标题、副标题、站点URL、语言、时区、关键词、站点启动日期 |
| 主题外观 | 视觉相关配置 | 主题色色相、主题色是否固定、字体配置（ASCII/CJK）、标签样式、壁纸模式 |
| 功能开关 | 各页面/功能启用状态 | 番剧、日记、友链、项目、技能、时间线、相册、设备页面的开关 |
| 导航配置 | 顶栏与侧边栏 | 导航链接（支持多级菜单）、侧边栏组件布局（左右侧分配、抽屉）、组件排序与动画延迟 |
| Banner 配置 | 顶部横幅 | 桌面/移动端图片源、轮播开关与间隔、水波纹效果、主页文字（标题/副标题/打字机效果）、图片API |
| 文章列表 | 列表页行为 | 默认布局模式（list/grid）、是否允许切换、分类导航条开关 |
| 第三方集成 | 外部服务配置 | Bangumi用户ID、Bilibili用户ID、评论系统（Twikoo/Giscus）配置、音乐播放器（Meting/本地）配置、Umami/Clarity统计 |
| 特效配置 | 交互特效 | 樱花特效参数（数量、速度、透明度）、Pio看板娘配置 |
| 其他功能 | 杂项开关 | 目录系统开关与深度、OG图片生成开关、页面进度条、显示上次编辑时间、分享功能、版权声明 |

**数据需求**

- 配置项按 key-value 形式存储，value 支持嵌套对象和数组
- 部分配置项支持多语言（如公告内容、Banner副标题）
- 配置变更后需能触发前端重新构建

---

### 2.2 About 页面内容管理

**功能说明**

About 页面为纯 Markdown 内容页面，当前存储在 `src/content/spec/about.md`，由 `about.astro` 页面渲染。后端化后，Markdown 内容需通过接口获取，前端在构建时渲染。

**数据需求**

- Markdown 原始文本内容
- 支持标准 Markdown 语法及项目自定义指令（如 `::github{repo="..."}`）
- 内容更新后需能触发前端重新构建

---

### 2.3 文件上传服务

**功能说明**

提供通用的文件上传能力，供各业务模块使用。当前通过 Gradio 直接操作文件系统，后端化后需统一上传接口。

**使用场景**

| 场景 | 目标目录 | 说明 |
|------|---------|------|
| 文章封面图 | posts | 与文章关联的封面图片 |
| 相册图片 | albums | 相册内照片，支持批量上传 |
| 日记配图 | diary | 日记条目中的图片 |
| 设备图片 | devices | 设备展示图 |
| 头像/Logo | avatars | 用户头像、站点Logo |
| 横幅/壁纸 | assets | Banner、全屏壁纸等主题资源 |

**数据需求**

- 支持单文件和多文件上传
- 上传后返回可访问的 URL
- 需记录原始文件名、存储路径、文件尺寸、MIME 类型
- 支持可选的图片后处理：格式转换（如转为 WebP）、缩略图生成

---

### 2.4 OG 图片生成

**功能说明**

根据文章标题、描述、发布日期等信息，动态生成 OpenGraph 分享图片（1200x630）。当前在构建时通过 `src/pages/og/[...slug].png.ts` 使用 Satori + Sharp 生成。后端化后，需支持运行时生成或预生成缓存。

**生成要素**

- 站点标题与图标
- 文章标题（最多3行，超出截断）
- 文章描述（最多2行，可选）
- 作者头像与名称
- 文章发布日期
- 主题色（基于站点配置的主题色色相）

**数据需求**

- 文章标题、描述、发布日期
- 站点配置：标题、主题色色相
- 作者信息：名称、头像
- 站点图标（favicon）
- 字体文件（用于中文/日文渲染）

---

### 2.5 RSS / Atom 订阅

**功能说明**

生成标准 RSS 2.0 和 Atom 1.0 订阅源，供读者通过 RSS 阅读器订阅博客更新。当前在构建时通过 `src/pages/rss.xml.ts` 和 `atom.xml.ts` 生成。

**数据需求**

- 站点信息：标题、副标题、站点URL、语言
- 文章列表（按发布日期排序，排除加密文章和草稿）：
  - 标题、描述、发布日期、更新日期、链接
  - 文章正文 HTML（Markdown 渲染后，图片路径需转为绝对 URL）
  - 作者名称
  - 分类/标签

---

### 2.6 站点统计信息

**功能说明**

提供站点维度的聚合统计数据，用于侧边栏「站点统计」小部件展示。

**统计项**

| 统计项 | 说明 | 计算方式 |
|--------|------|---------|
| 文章总数 | 已发布文章数量 | 排除草稿 |
| 分类总数 | 不重复的分类数量 | 基于文章分类字段聚合 |
| 标签总数 | 不重复的标签数量 | 基于文章标签字段聚合 |
| 总字数 | 所有文章正文字数合计 | CJK 字符逐个计数，非 CJK 按单词计数，排除代码块 |
| 运行天数 | 站点自启动日期至今的天数 | 基于 `siteStartDate` 计算 |
| 最后更新 | 距最新文章发布日的天数 | 基于最新文章发布日期计算 |

**数据需求**

- 所有已发布文章的完整正文内容（用于字数统计）
- 文章分类、标签字段
- 站点启动日期配置

## 4. 前端消费方式

### 4.1 构建时获取（SSG）

所有通用模块数据在 Astro 构建时通过接口获取，嵌入到静态 HTML 中：

- 站点配置：构建前一次性获取，注入到全局上下文
- About 内容：构建时获取 Markdown，通过 remark/rehype 管线渲染为 HTML
- RSS/Atom：构建时生成 XML 文件
- OG 图片：构建时生成 PNG 图片（或运行时通过 API 生成）
- 站点统计：构建时计算并嵌入 HTML

### 4.2 客户端交互

以下场景需在客户端动态获取或更新：

- 文件上传：CMS 后台或前端交互中上传图片
- OG 图片：如改为运行时生成，客户端通过 URL 参数请求

---

## 5. 数据迁移要点

| 当前存储 | 当前位置 | 后端化方向 |
|---------|---------|-----------|
| 站点配置 | `src/config.ts` | 配置表 / 配置服务 |
| About 内容 | `src/content/spec/about.md` | 内容管理接口 |
| 图片文件 | `public/images/`、`public/assets/` | 对象存储 + 上传接口 |
| OG 图片 | 构建时生成到 `dist/` | 预生成缓存或运行时生成 |
| RSS/Atom | 构建时生成到 `dist/` | 构建时通过接口数据生成 |
| 站点统计 | 构建时计算 | 构建时通过接口数据计算 |

---

## 6. 多语言支持

当前站点通过 `siteConfig.lang` 指定单一语言，i18n 翻译文件为前端静态 TS 对象。后端化后需注意：

- 站点配置中的语言字段决定前端使用哪套翻译
- 部分配置项（如公告内容、Banner副标题）可能随语言变化
- RSS/Atom 的 `<language>` 标签需与站点语言一致
